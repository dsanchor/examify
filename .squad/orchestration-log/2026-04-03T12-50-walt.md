## Design Decisions

1. **Monorepo Architecture**: Single repo with `client/` and `server/` workspaces for easy deployment
2. **Service Layer Pattern**: Keep routes thin, business logic in testable services
3. **Shared TypeScript Types**: Define models once in server, mirror in client for API consistency
4. **Chapter Extraction**: AI instructed NOT to infer chapters (only extract explicitly labeled ones)
5. **Answer Padding**: Handled during exam generation, not extraction
6. **Test Modes**: Support both all-at-once (traditional) and one-by-one (learning) modes
7. **Multi-stage Docker**: Efficient production image serving static frontend + Node.js backend

## Key Implementation Details

- **AI Extraction**: PDF text extraction → structured prompt → JSON parsing with validation
- **Exam Generation**: Question filtering, shuffling, answer padding with intelligent fillers
- **Test Sessions**: Track answers, auto-save, optional countdown timer, detailed scoring
- **Frontend**: SPA with React Router, custom timer hook, typed Axios API client, form validation
- **Error Handling**: Centralized Express middleware, custom AppError class, user-friendly messages
